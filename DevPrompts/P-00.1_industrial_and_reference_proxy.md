## Контекст и major change vs P-00.1 v1

P-00.0 завершён. Step 0 (V1 archival) завершён. Repository есть. Common Plume Schema работает. GEE Asset folders structure инициализирована.

**MAJOR UPDATE (CHANGE-0017):** P-00.1 теперь имеет **dual scope** — не только industrial proxy mask, но и **protected areas reference mask**.

**Почему изменение:** методологическое улучшение от пользователя 2026-04-26. Industrial buffer exclusion (negative space approach) имеет fundamental weakness — определяет clean as `NOT(known industrial)` с unknown unknowns. Российские заповедники с federal protected status (IUCN Ia category) дают enforced clean baseline (positive space). Detection algorithm теперь использует dual baseline approach (Algorithm v2.3 §3.4).

**Без обоих масок (industrial + reference) Phase 1 не может стартовать:**
- Industrial mask → используется в regional climatology (P-01.0b) для buffer exclusion
- Reference protected areas mask → используется в reference baseline (P-01.0a) для positive space construction

- Centroid: ~51.5°N, 88.5°E

- Включается в production baseline ТОЛЬКО если QA test passes (см. §11.4 Algorithm)

**Где брать polygon boundaries:**
- **Primary source — OOPT info**: oopt.info часто имеет KML/GeoJSON downloads
- **WDPA (World Database on Protected Areas)**: protectedplanet.net — international database, имеет boundaries для российских заповедников. Может быть accessible через API или manual download. **License:** non-commercial, attribution required.
- **OpenStreetMap**: для некоторых заповедников OSM имеет полные boundaries (boundary=protected_area)
- **Минприроды РФ публикует kadastr** — некоторые boundaries есть в государственном кадастре особо охраняемых природных территорий

**Verification stages (для каждой zone):**
1. Total area из GeoJSON ≈ official area (within 5%)
2. Centroid lat/lon ≈ documented values
3. Visually на карте — boundary makes sense (например, для Юганского — bassin рек Малый Юган и Негусъях)
4. NO industrial settlements внутри boundary (для verification — overlay GPPD + manual points)

**Если не можешь найти reliable polygon source — эскалируй:**
- Возможно потребуется manual digitization из карт
- Возможно потребуется academic source (научные публикации с boundaries)
- В худшем случае — approximate bounding box с warning

- Не можешь найти reliable polygon source для одного из заповедников

- Web search для координат вернул противоречивые результаты
- VIIRS threshold value требует sanity check
- Какой-то source_type не покрывается existing schema

Если найдёшь industrial points (manually добавленные или GPPD) близко к границе Юганского (within 30 km buffer) — это **expected**, не ошибка. Они должны быть **снаружи** boundary, но могут быть в внешней охранной зоне (Юганский имеет 2 km external охранную зону, плюс далее идёт промышленность).
