## TD-0021 — Zone-boundary smoothing parameters

**Issue:** Reference baseline shows discrete plateaus at latitude band boundaries. Step changes create spurious `consistency_flag=false` near boundaries (Algorithm §3.4.3).

**Quantified step sizes (75°E transect, 0.1° resolution, ±2° windows):**

| Boundary | Lat | step_M07 (ppb) | step_M10 (ppb) | n_above | n_below |
|----------|-----|----------------|----------------|---------|---------|
| Kuznetsky → Yugansky | 57.5°N | **+34.85** | +19.83 | 20 | 20 |
| Yugansky → Verkhne-Tazovsky | 62.0°N | **−16.18** | +1.82 | 20 | 20 |

**Phase 2A mitigation parameters:**

```python
# config['detection']['zone_boundary_smoothing']
{
    "enabled": True,
    "smoothing_method": "moving_average",
    "lat_window_deg": 2.0,  # ±1° around each boundary
    "boundaries_lat_n": [57.5, 62.0],
    "trigger_distance_km_to_boundary": 100,  # apply ONLY when candidate within 100 km
    "rationale": (
        "Step sizes 34.85 ppb (57.5N M07) и 16.18 ppb (62N M07) exceed "
        "consistency_tolerance_ppb=30 default — без smoothing, candidates "
        "near boundaries trigger false consistency_flag=false."
    ),
}
```

**Implementation hint:** `src/js/modules/reference_baseline.js` → add `applyBoundarySmoothing(refImage, boundaries, windowDeg)` helper. Apply `focal_mean(latitude window)` only inside ±100 km of each boundary, leave away-from-boundary pixels untouched.

**Future improvement (deferred к CHANGE-0018 candidate):** distance-weighted blend nearest-2 zones instead of step-stratification. Smoothing is interim mitigation.

## TD-0019 — Confidence factor formulation (distance-from-zone)

**Investigation result (P-01.0b extrapolation, R²=0.002):** Distance к nearest zone centroid is **NOT predictive** of |Δ|. Confidence-factor formulation should reflect this empirically.

**Phase 1c reconfirmation:** mean |Δ| globally = 19.42 ppb (M07), 14.49 ppb (M10). Variance bounded within ±~50 ppb. No distance-driven systematic bias detected.

**Recommended Phase 2A confidence factor:**

```python
# config['detection']['confidence_factor']
{
    "distance_factor": {
        "method": "constant",
        "value": 1.0,
        "rationale": "TD-0019 R²=0.002 — distance к centroid не predicts |Δ|.",
    },
    "baseline_consistency_factor": {
        "method": "linear_penalty_for_extreme_distance",
        "lat_dist_km_threshold": 1000,  # below this, no penalty
        "lat_dist_km_max": 1500,  # above this, factor → 0.5 (capped)
        "formula": "max(0.5, 1.0 - max(0, (lat_dist_km - 1000) / 1000))",
        "rationale": (
            "Distance не predicts |Δ| empirically; soft penalty applied только "
            "for extreme cases (>1000 km from any zone centroid — only at far "
            "northern AOI edge). Bounded floor 0.5 prevents detection drop-off."
        ),
    },
}
```

## Caveats

1. **Moran's I = 0.736 (high spatial autocorrelation, p=0.001)** — reflects dual baseline architecture's inherent properties (zone band step changes + clustered suspect regions), не a methodology bug. Document как expected feature in tool-paper.
2. **Hartigan's dip p=0.035 (M07)** — borderline bimodal distribution. Consistent с two-population mixture (zone-aligned pixels vs suspect-cluster pixels). Acceptable but worth noting in tool-paper discussion.
3. **Latitude transect at single longitude (75°E)** — boundary step quantification used 75°E only. Longitude-dependence of boundary effect не measured. Future deeper analysis может sample 3-4 longitudes для more robust step quantification.
4. **Fine transect resolution (0.1°)** — 251 points may miss sub-pixel structure. Sufficient для detecting step changes but не для retrieving spatial gradients.
